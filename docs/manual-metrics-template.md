# Manual Metrics Template

Use this as a daily copy/paste table while the product is still founder-led. Keep the numbers manual from Payhip, optional no-PII ping logs, support inbox notes, and social posts. Do not build a dashboard until the same measurement need repeats for several weeks.

## How To Fill It

- Landing visits: Payhip/product page visits, Vercel analytics if already available, or manual link-click estimates from the channel.
- Form submits: optional `portal_form_submit` ping count if `NEXT_PUBLIC_PING_URL` is configured.
- Purchases: completed Payhip orders.
- Portal opens: optional `portal_link_opened` ping count if configured.
- Support issues: short labels such as `lost link`, `checkout handoff`, `timezone`, `refund`, `copy unclear`.
- Notes: quote patterns, objections, geographies, devices, or parent-group context.
- Next action: one concrete action for tomorrow.

## Weekly Review

At the end of each week, answer:

1. Which traffic source produced the most real conversations?
2. Which source produced purchases, not just compliments?
3. Which support issue repeated?
4. Which geography or community understood the product fastest?
5. What copy should change before more outreach?
6. What should stay manual for now?
